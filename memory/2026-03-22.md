## 今日大事

### 修复了三个定时任务发不到飞书的问题 🦞

**问题症状：**
- 昨天设置的每日美文/数学/英语定时任务，一直发不到飞书群
- 飞书 main 机器人状态显示 running，但群里收不到消息

**根本原因：**
- 三个 job 全部配置为 `sessionTarget: "isolated"`（独立 session 执行）
- isolated session **没有 exec 权限**，无法运行 `openclaw message send` 命令
- AI 改用 feishu_chat 工具时参数不对，报 HTTP 400 错误
- job 最终超时失败

**修复方案：**
- 三个 job 全部从 `isolated` 改为 `main` + `systemEvent`
- 这样任务在主 session 执行，有完整 exec 权限

**经验教训已写入 MEMORY.md 的「Cron Job 最佳实践」章节，后续不会再犯**
